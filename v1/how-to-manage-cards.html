Managing cards on the front page

Cards are plain HTML blocks in film4ever.html — no CMS, no build step. Open the file in any text editor, find the block you want to change, edit and save. This page explains every operation with the exact HTML patterns to use.

card anatomy

Every card on the front page is one self-contained HTML block. Published cards use <a> (a link); coming-soon cards use <div> (not clickable). Both share the same internal structure — four editable parts.

published card — full structure
<!-- 1. outer element: <a> for published, <div> for coming-soon -->
<a class="graph-card" href="graphN.html">

  <div class="card-head">
    <div>
      <div class="card-id">graph N</div>          <!-- 2. id label -->
      <div class="card-title">Title <em>&rarr;</em> Subtitle</div>  <!-- 3. title -->
    </div>
    <div class="card-arrow">&rarr;</div>   <!-- arrow (published only) -->
  </div>

  <div class="card-body">
    <p class="card-desc">Description text here.</p>   <!-- 4. description -->
    <div class="card-tags">
      <span class="tag">concept one</span>         <!-- 5. tags -->
      <span class="tag">concept two</span>
    </div>
  </div>

</a>
tip — italic arrows in titles
Use <em>&rarr;</em> between concepts in the title to get the styled italic arrow: →. Use &amp; for an ampersand.
operations
Add Add a published card

Copy the block below and paste it inside the .cards-grid div under the published <section>. Edit the five highlighted parts.

1Find the comment <!-- published --> in film4ever.html
2Paste a new <a class="graph-card"> block inside its .cards-grid
3Set href, card-id, card-title, card-desc, and tag spans
4Save — grid lays out automatically, two columns
Add Add a coming-soon card

Same structure but three differences: outer tag is <div>, add class soon, replace the arrow with a badge.

coming-soon card — differences only
<div class="graph-card soon">
  <div class="card-head">
    <div>
      <div class="card-id">graph N</div>
      <div class="card-title">...</div>
    </div>
    <span class="soon-badge">soon</span>
  </div>
  <div class="card-body">...</div>
</div>
Promote Promote coming-soon → published

When a graph page is ready to publish, four edits turn its coming-soon card into a live published card.

1Change opening tag: <div class="graph-card soon"> → <a class="graph-card" href="graphN.html">
2Change closing tag: </div> → </a>
3Replace <span class="soon-badge">soon</span> with <div class="card-arrow">&rarr;</div>
4Cut the whole block from the coming soon section and paste it into the published section's .cards-grid
Remove Remove a card

Delete the entire block — from the opening tag to its matching closing tag. The grid reflows automatically.

1Find the card by its card-id text (e.g. "graph 3")
2Select from <a class="graph-card" (or <div class="graph-card) all the way to the matching </a> or </div>
3Delete — remaining cards fill the grid
watch for nesting
Each card contains multiple nested <div> tags. Make sure you delete from the outermost opening tag to the outermost closing tag, not just an inner one.
Edit Edit card content — field by field
Field Class / element Notes Operation
Link target href="..." on the <a> Filename of the graph page, e.g. graph1.html edit
ID label .card-id Short identifier shown in accent red above the title. E.g. graph 1 edit
Title .card-title Use <em>&rarr;</em> for italic arrows between concepts edit
Description .card-desc 2–3 sentences. Keep under ~180 characters for visual consistency edit
Tags <span class="tag"> Each tag is independent. Add or remove any number — they wrap automatically add remove edit
layout control

The grid is always two columns. Cards fill left-to-right, top-to-bottom. An odd number of cards leaves the last one in the left column. No configuration needed — it is automatic.

Layout Make a card span both columns

Add an inline style to the card's opening tag. Useful for a featured or intro card at the top of a section.

<a class="graph-card"
   href="graphN.html"
   style="grid-column: 1 / -1">
Layout Add a new section

Sections (published, coming soon) are <section class="series-section"> blocks. Copy one and change the label text.

<section class="series-section">
  <div class="section-label">your label</div>
  <div class="cards-grid">
    <!-- cards go here -->
  </div>
</section>
quick reference — class names
Class Element Purpose
graph-card<a> or <div>Outer wrapper for any card
graph-card soon<div>Coming-soon card — dimmed, not clickable
card-head<div>Top section of card containing id + title + arrow/badge
card-id<div>Accent-coloured label above the title
card-title<div>Serif display title. Wrap with <em> for italic portions
card-arrow<div>Right-side arrow on published cards. Contains &rarr;
soon-badge<span>Right-side "soon" label on coming-soon cards. Replaces arrow
card-body<div>Lower section containing description + tags
card-desc<p>Description paragraph
card-tags<div>Container for concept tags
tag<span>Individual concept tag pill
cards-grid<div>Two-column grid container. Place cards directly inside this
series-section<section>Section wrapper with consistent padding
section-label<div>Section heading with auto rule line on the right
tip — testing locally
The vtt-branding.js loader uses fetch() which requires HTTP — it won't run when you double-click an HTML file. To preview locally, run python3 -m http.server 8000 in the folder and open localhost:8000/film4ever.html in your browser.
